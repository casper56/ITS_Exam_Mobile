Azure AI 語言服務 (Azure AI Language) 總覽

Azure AI 語言 (Azure AI Language) 是微軟 Azure 提供的雲端自然語言處理 (NLP) 服務。它將過去分散的語言理解 (LUIS)、QnA Maker 與文字分析服務整合為單一的強大 API，讓開發者能輕鬆將自然語言的「理解」與「分析」能力加入應用程式中。

此服務主要分為三大類別：開箱即用的預建功能、可量身打造的自訂功能，以及專精領域的醫療保健功能。

一、 預建功能 (Pre-configured Features)

這些功能已經過微軟大量資料的預先訓練，不需準備訓練資料即可直接呼叫 API 使用：

  • 情感分析與意見探勘 (Sentiment Analysis & Opinion Mining)：
    分析一段文字的情緒是正面、負面還是中立（可細分到句子層級）。「意見探勘」更能針對特定目標給出情緒，例如：「這間飯店的床很舒服 (正面)，但服務很差 (負面)」。
  • 關鍵片語擷取 (Key Phrase Extraction)：
    快速從長篇文章中抓出核心重點詞彙，幫助辨識討論的主要議題。
  • 命名實體識別 (NER, Named Entity Recognition)：
    自動辨識並分類文字中的實體，例如：人名、地名、組織機構名稱、日期、數量等。
  • 個人識別資訊 (PII) 偵測與遮蔽：
    找出文字中的敏感個資（如身分證字號、信用卡號、電話、Email）並自動將其遮蔽（例如替換成 ***），這對資安與合規性非常重要。
  • 語言偵測 (Language Detection)：
    自動判斷輸入的文字是哪一種語言，並給出信心分數（支援超過 100 種語言）。
  • 文字摘要 (Text Summarization)：
  • 抽取式摘要： 挑選文章中最重要的原句組合段落。
  • 生成式摘要： 使用 AI 重新組織並用新的句子寫出精簡的總結。

二、 自訂功能 (Customizable Features)

當預建模型無法滿足特定產業的專有詞彙或特定對話情境時，您可以提供自己的資料來訓練專屬模型：

  • 交談式語言理解 (CLU, Conversational Language Understanding)：
    (這是過去 LUIS 服務的升級版) 專門用於打造聊天機器人或語音助理。它可以從使用者的對話中抓出意圖 (Intent)（例如：訂機票）與實體 (Entity)（例如：出發地「台北」、目的地「東京」）。
  • 自訂問題解答 (Custom Question Answering)：
    (這是過去 QnA Maker 的升級版) 只要上傳企業內部的 FAQ 文件、PDF 手冊或網頁連結，它就能自動萃取問題與答案，快速生成一個專屬的知識庫問答機器人 (QA Bot)。
  • 自訂文字分類 (Custom Text Classification)：
    將文本自動分類到您自訂的類別中。包含「單一標籤」（一篇文章只能有一個分類）與「多重標籤」（一篇文章可同時屬於多個分類，如同時是"政治"與"經濟"）。
  • 自訂命名實體識別 (Custom NER)：
    如果您的領域有非常特殊的專有名詞（例如：特定型號的電子零件、特殊的法律術語），可以自行標註資料來訓練 AI 認得這些特殊實體。
  • 協調流程 (Orchestration Workflow)：
    當您有多個不同的 CLU 或 QA 模型時，這個功能可以作為「大腦路由器」，判斷使用者的輸入應該交給哪一個子模型去處理。

三、 專精領域功能

  • 醫療保健文字分析 (Text Analytics for Health)：
    這是一個專門為醫療領域打造的進階功能。它可以從非結構化的醫療紀錄、醫生筆記中，提取出疾病、症狀、藥物、劑量等專業醫學實體，並能找出它們之間的關聯（例如：某藥物與某劑量的關聯），甚至判斷這些症狀是否被否定（例如：「沒有咳嗽症狀」）。

總結應用場景

結合這些服務，企業可以打造出：智慧客服機器人（CLU + 問題解答）、社群輿情監控系統（情感分析 + 關鍵片語）、合規與個資過濾系統（PII 偵測）、以及自動化文件路由系統（自訂文字分類）。
